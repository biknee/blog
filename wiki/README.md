# 项目知识库

知识库用于辅助 Claude 在本项目中高效工作，减少重复扫描和重复犯错。

## 作用

- **节约 Token**：需要全局信息的操作优先查知识库，查不到再去扫描源码
- **避免重犯**：记录 Claude 在本项目犯过的错误及修复方式，遇到同类问题先查此处
- **防止冲突**：新增功能前先查看已有功能清单，发现疑似重复及时向用户确认
- **快速定位**：记录每个文件/模块的职责和关键符号，避免反复 `grep` 和通读文件

## 可纳入知识库的场景

以下动作消耗 Token 较多或容易重复，应优先查知识库：

| 场景 | 说明 |
|------|------|
| 定位功能代码 | 先查架构文档，避免全局 `grep` 扫描 |
| 新增功能 | 先查功能清单，确认是否已存在或冲突 |
| 遇到报错 | 先查已知错误，避免重复排查 |
| 修改样式/模板 | 先查设计约定，避免风格不一致 |
| 添加依赖 | 先查 go.mod 快照（在架构文档中），确认是否已有 |

## 更新规则

- 涉及知识库覆盖内容的代码变更时，**同步更新**对应知识文件
- 旧知识完全过时时**及时删除**，避免误导
- 同类新知识以**追加形式**写入已有文件，不新建碎片文件
- 文件粒度：按功能模块划分，既不分得过细（避免文件过多），也不合并成大文件（避免难以定位）
